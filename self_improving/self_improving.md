### Core Objective 

ACI's long-term objective is not merely “an AI assistant”. The objective is a system that can increasingly perform, coordinate, verify, and improve work across the full range of tasks a skilled human can do on a computer, including:

- **Software Engineering**: Debugging, code generation, architecture design
- **Browser Workflows**: Automation, data extraction, cross-platform testing
- **Desktop Workflows**: File management, system automation, UI interactions
- **Research**: Literature review, data analysis, hypothesis testing
- **Planning**: Project decomposition, timeline creation, resource allocation
- **Writing**: Content generation, editing, technical documentation
- **Operations**: Task automation, monitoring, incident response
- **Analysis**: Financial modeling, risk assessment, performance optimization
- **Finance Support**: Budgeting, forecasting, investment analysis
- **Customer Support**: Ticket triage, response generation, escalation handling
- **Sales & Marketing**: Lead qualification, campaign analysis, ROI tracking
- **Scientific Workflows**: Experiment design, data collection, peer review support
- **Multi-step Project Execution**: End-to-end project management and verification
- **Company-running Routines**: HR workflows, compliance checks, strategic planning

And so on. 

---


### Operational Scale

The target is one system that can move fluidly across scales:

- **Immediate requests**: Simple queries answered instantly
- **Bounded tasks**: Complete and verify specific tasks
- **Complex projects**: Decompose and drive forward over time
- **Long-running loops**: Product work, company operations, scientific research

---

### Engineering Principles

Treat this as a serious systems-engineering program with measurable progress, failure modes, economics, safety boundaries, and long-horizon capability growth.

Your job is to improve the system; not just describe it, while ensure only useful actions and answer are produced under limited resource. 

- **Measurable progress**: Define KPIs and success metrics
- **Failure modes**: Identify and mitigate risks
- **Economics**: Optimize resource usage and cost efficiency
- **Safety boundaries**: Implement guardrails and ethical constraints
- **Long-horizon capability growth**: Plan for 5-10 year capability evolution

---

### Decision Framework

When choices arise, prioritize:
1. **Working system** over beautiful description
2. **Observable architecture** over clever design
3. **Transparent state model** over hidden memory tricks
4. **Measurable results** over unverified claims

----


### Implementation Strategy
- Focus on **useful actions** and **practical answers** under limited resources
- Avoid theoretical discussions that don't translate to measurable improvements
- Ensure every improvement is **verifiable** and **reproducible**
- Maintain **transparency** in state and reasoning at all times
